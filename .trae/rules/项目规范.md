# Midnight Atelier 项目规范

## 项目概述

Midnight Atelier（室内大师）是一个 AI 图片生成应用，支持多种视觉预设风格、画廊社区、账户管理等功能。

## 技术栈

### 前端
- **框架**: React 18 + TypeScript
- **构建工具**: Vite 6
- **样式**: Tailwind CSS 4
- **动画**: motion/react (framer-motion)
- **图标**: lucide-react
- **3D 渲染**: Three.js
- **图像处理**: Fabric.js

### 后端/部署
- **Serverless Functions**: Netlify Functions / Vercel
- **存储**: 阿里云 OSS

## 代码规范

### 命名约定

1. **组件命名**: 使用 PascalCase
   ```tsx
   // 正确
   export const Dropdown: React.FC<DropdownProps> = () => {}
   export default function App() {}

   // 错误
   export const dropdown = () => {}
   ```

2. **文件命名**:
   - 组件文件: PascalCase (如 `Dropdown.tsx`, `ImageEditor.tsx`)
   - 工具函数: camelCase (如 `imageUtils.ts`, `download.ts`)
   - 类型文件: camelCase (如 `generation.ts`)

3. **变量和函数**: 使用 camelCase
   ```typescript
   const selectedPreset = useState();
   const showToast = useCallback(() => {});
   ```

4. **常量**: 使用 UPPER_SNAKE_CASE
   ```typescript
   const MAX_IMAGE_SIZE = 4096;
   ```

5. **类型和接口**: 使用 PascalCase
   ```typescript
   interface DropdownOption {
     value: string;
     label: React.ReactNode;
   }
   type MenuItemId = string;
   ```

### TypeScript 规范

1. **显式类型定义**: 为 props、state 和函数返回值定义类型
   ```tsx
   interface ToastMessage {
     id: string;
     type: 'success' | 'error' | 'info';
     message: string;
   }
   ```

2. **避免使用 any**: 使用 unknown 或具体类型
   ```typescript
   // 正确
   const handleSelect = (value: string) => {};

   // 避免
   const handleSelect = (value: any) => {};
   ```

3. **使用类型推断**: 简单类型无需显式声明
   ```typescript
   // 正确
   const [isOpen, setIsOpen] = useState(false);

   // 不必要
   const [isOpen, setIsOpen] = useState<boolean>(false);
   ```

### React 规范

1. **函数式组件**: 统一使用函数式组件和 Hooks
   ```tsx
   // 推荐
   export const Dropdown: React.FC<DropdownProps> = ({ options, value }) => {
     const [isOpen, setIsOpen] = useState(false);
     // ...
   };

   // 或
   export default function App() {
     // ...
   }
   ```

2. **事件处理函数**: 使用 `handle` 前缀
   ```tsx
   const handleSaveApiKey = () => {};
   const handleMenuItemClick = (item: MenuItem) => {};
   ```

3. **回调函数优化**: 使用 useCallback 避免不必要的重渲染
   ```tsx
   const showToast = useCallback((type: 'success' | 'error' | 'info', message: string) => {
     // ...
   }, []);
   ```

4. **条件渲染**: 使用短路求值或三元运算符
   ```tsx
   // 简单条件
   {isLoading && <LoadingSpinner />}
   {hasApiKey ? <Dashboard /> : <Login />}
   ```

### 样式规范

1. **Tailwind CSS**: 优先使用 Tailwind 工具类
   ```tsx
   <button className="px-4 py-2 bg-indigo-500 text-white rounded-lg hover:bg-indigo-600 transition-colors">
     提交
   </button>
   ```

2. **CSS 变量**: 使用项目定义的语义化变量
   ```css
   /* 推荐使用的变量 */
   --text-primary
   --text-secondary
   --text-muted
   --bg-subtle
   --surface-1
   --surface-2
   --surface-3
   --border
   --border-subtle
   --c-shadow-heavy
   ```

3. **自定义类名**: 使用语义化命名
   ```tsx
   // 推荐
   className="glass-card"
   className="input-field"
   className="btn-primary"
   className="custom-scrollbar"
   ```

4. **响应式设计**: 移动端优先
   ```tsx
   // 使用 useMobile hook
   const isMobile = useMobile();

   // 条件渲染
   <div className={`${isMobile ? 'px-3' : 'px-8'}`}>
   ```

### 动画规范

1. **使用 motion/react**:
   ```tsx
   import { motion, AnimatePresence } from 'motion/react';

   // 入场动画
   <motion.div
     initial={{ opacity: 0, y: 10 }}
     animate={{ opacity: 1, y: 0 }}
     exit={{ opacity: 0, y: -10 }}
     transition={{ duration: 0.2 }}
   >
   ```

2. **列表动画**: 使用 AnimatePresence 处理进入/退出
   ```tsx
   <AnimatePresence>
     {toasts.map(toast => (
       <Toast key={toast.id} toast={toast} />
     ))}
   </AnimatePresence>
   ```

## 注释规范

1. **文件头注释**: 使用 SPDX 许可证标识
   ```tsx
   /**
    * @license
    * SPDX-License-Identifier: Apache-2.0
    */
   ```

2. **函数注释**: 复杂逻辑添加说明
   ```typescript
   /**
    * 更新下拉面板位置，确保不超出视口
    */
   const updatePosition = useCallback(() => {
     // ...
   }, [isOpen, direction]);
   ```

3. **行内注释**: 说明"为什么"而非"是什么"
   ```typescript
   // 防止面板超出视口
   const maxW = Math.min(rect.width, vw - 16);
   ```

## Git 提交规范

使用约定式提交:

- `feat`: 新功能
- `fix`: Bug 修复
- `refactor`: 代码重构
- `style`: 代码格式调整
- `docs`: 文档更新
- `chore`: 构建/工具变更

示例:
```
feat: 添加图片预览功能
fix: 修复下拉菜单定位问题
refactor: 重构 API 调用逻辑
```

## 禁止事项

1. **不要使用 emoji** 在代码和注释中（除非用户明确要求）
2. **不要使用 `any` 类型**，使用具体类型或 `unknown`
3. **不要在组件中直接操作 DOM**，使用 React ref
4. **不要在渲染函数中定义新组件**，提取到组件外部
5. **不要忽略 ESLint 警告**，修复或合理禁用

## 最佳实践

1. **组件拆分**: 保持组件职责单一，复杂组件拆分为子组件
2. **状态提升**: 共享状态放在最近的共同父组件
3. **性能优化**: 使用 `useCallback`、`useMemo` 避免不必要渲染
4. **错误处理**: 使用 ErrorBoundary 捕获组件错误
5. **可访问性**: 为交互元素添加 `aria-*` 属性和 `type="button"`
